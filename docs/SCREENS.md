# SKAI 화면 캡처 — 기능별 시각 문서

> **한 줄**: `scripts/demo.sh replay` 기동 화면을 기능별로 캡처해 각 스크린샷이 어떤 온톨로지 요소의
> 렌더인지 짚어둔 문서. 사용법 전반은 [USER-GUIDE.md](USER-GUIDE.md), 프로젝트 개요는
> [README.md](../README.md) 참조.

## 기준 모드

아래 캡처는 전부 **재생(replay) 모드**(`scripts/demo.sh replay`) 기준입니다 — 네트워크 0·결정적
합성 데이터 13건(비상 스쿽·ADS-B dropout·로이터링·군용기 접근·위성 근접·급기동)이 고정 스냅샷으로
주입된 상태입니다. 헤더의 노란 **"재생 모드 · 고정 스냅샷"** 배지와 각 항목의 **[합성]** 배지가 그
표식입니다. **실데이터 live 모드**(`scripts/demo.sh live`)도 동일한 화면 구조·패널 배치를 그대로
쓰며, 차이는 데이터 출처(실 API 대 고정 스냅샷)와 상단 LIVE 배지뿐입니다.

## 00. 온보딩 오버레이

![](screens/00_onboarding.png)

첫 방문(또는 "다시 보지 않기" 미체크) 시 뜨는 안내 오버레이. 화면의 4개 구역(①지도 ②이상징후
타임라인 ③소스·기상·뉴스 ④코파일럿)과 핵심 동선 3가지(질문→근거 답변, 이상징후 결정 루프,
은닉 정황→서브그래프)를 번호로 매핑해 설명합니다. 하단 **정직 안내** 박스가 "이상탐지는 규칙 기반,
서술은 템플릿(옵션 LLM 다듬기)"·"평가 정밀도는 자작 합성 시나리오 기준(라이브 실측 아님)"을 명시 —
과장 없는 정직 표기입니다. 딥링크(`?q=` 또는 `?anom=`)로 진입하면 이 오버레이는 자동 스킵됩니다.

## 01. 전체 개요 (3열 레이아웃)

![](screens/01_overview.png)

온보딩을 닫은 직후의 기본 화면. 좌측 이상징후 타임라인(13건, 유형별 아이콘·색), 중앙 지도(KADIZ
작전구역·항적·군용 추정 항적·위성 지상궤적), 우측 소스/기상/뉴스 패널, 하단 코파일럿 도크가 한
화면에 조립됩니다. 지도 위 이상징후 마커는 `Anomaly` 온톨로지 객체, 좌측 목록 항목은 같은 객체를
시간순으로 렌더한 것 — 클릭하면 지도 포커스·근거 흐름선·상세 카드가 동시에 갱신됩니다.

## 02. 범례 · 레이어 토글

![](screens/02_legend_layers.png)

지도 좌상단 범례(기본 펼침 상태) — 항적(콜사인)·군용 추정 항적(저신뢰)·트랙 경로·ADS-B gap
트랙·위성 지상궤적·이상징후(실데이터/합성 구분 점선)·작전구역 기호, 그리고 이상징후 유형별 색상
칩. 하단 3종 토글(항공기 표시·위성 궤적 표시·실시간 잔상 — "전체 트랙"까지 포함하면 4종)은 지도
레이어의 on/off 스위치로, 민간 항공기 레이어와 위성 궤적 레이어를 독립적으로 끄고 켤 수 있습니다
(온톨로지 객체 자체는 유지되고 지도 렌더만 토글).

## 03. 이상징후 상세 — 비상 스쿽(CES7500)

![](screens/03_anomaly_evidence.png)

가장 높은 신뢰도(0.95)의 이상징후. 설명문(비상 스쿽 7500·불법 간섭/하이재킹 코드)과 **근거
(EVIDENCE·PROVENANCE)** 카드가 좌측 상세 패널에 렌더됩니다. 카드는 `Anomaly` 객체의
`evidenced_by` 링크를 따라간 `Observation` 객체(콜사인·스쿽·관측 시각·출처 URL)를 그대로
보여주는 것 — 근거 없는 문장은 애초에 조립되지 않는다는 프로젝트 원칙(provenance 강제)의 시각적
증거입니다. 아직 사람이 검토하지 않은 **"후보"** 상태(주황 배지)라 확인/기각 버튼이 활성 상태입니다.

## 04. 교차소스 상관 — ADS-B dropout(SHADOW7)

![](screens/04_correlation_reason.png)

작전구역 내 ADS-B 신호 단절(신뢰도 0.72) 이상징후. 설명문에 2차 소스(미러) 교차확인 사실이
명시되고, 아래 **교차소스 상관(CORRELATED_WITH)** 섹션에 다른 `Anomaly` 객체와의 관계가 근거
문장으로 풀립니다 — 예: "이상징후와 시간차 5분 · 거리 158.2 km". 이 문장은 `correlated_with`
링크의 `attrs`(시간차 `gap_s`, 거리 `distance_km`)를 그대로 자연어로 조립한 것으로, "왜 이 둘이
엮였는지"를 추측이 아니라 링크 속성으로 답합니다.

## 05. 군용기 접근 — PLAAF01 (합성 시나리오)

![](screens/05_military_synth.png)

서해 작전구역 진입 이상징후. 설명문 앞의 **`[합성 시나리오]`** 태그는 실 ADS-B로 관측 불가능한
군용기 시나리오임을 명시하는 정직 표기이고, "저신뢰 휴리스틱(신뢰도 0.55)"·"군용 판정은
콜사인·대역 기반으로 오탐 가능 — 교차검증 요망" 문구는 이 판정의 한계를 스스로 고지합니다. 실제
군용 식별은 관측 소스의 `is_military` 플래그(콜사인 프리픽스·대역 휴리스틱, live 모드에선
adsb.fi 2차 보강)에서 온 것이지 확정 판정이 아닙니다.

## 06. 코파일럿 — "지금 KADIZ 근방 이상한 거 있어?"

![](screens/06_copilot_cites.png)

자연어 질의에 대한 코파일럿 응답. 상단 **투명성 스트립**(의도·슬롯·분류·지역·창·히트)이 이 질의가
어떻게 해석됐는지 먼저 밝히고, 본문 헤드라인과 이상징후별 문장마다 파란 **cites 배지**가 붙습니다.
각 배지는 `SituationAssessment` 객체가 인용한 `Anomaly`/`Observation` 객체로 직접 연결되는
링크 — "문장마다 근거 강제, 근거 없는 주장은 출력 안 함" 원칙이 화면에 드러나는 지점입니다.

## 07. 코파일럿 — "군용기만 보여줘" (필터 의도)

![](screens/07_copilot_filter.png)

같은 코파일럿이지만 의도 분류가 **필터**(`military=true`)로 잡혀, Q&A 서술이 아니라 조건에 맞는
항적 목록(ROKAF31·PLAAF01·OLIVE21·RCH451·FALCON9)을 직접 반환합니다. 투명성 스트립의
"의도 필터 · 슬롯 military=true · 분류 rule: True"가 질의가 요약이 아니라 구조화된 필터 쿼리로
처리됐음을 보여줍니다.

## 08. 온톨로지 서브그래프

![](screens/08_subgraph.png)

"서해 쪽 기상이랑 뉴스 맥락 요약해줘" 질의가 만든 `SituationAssessment` 객체(중앙 초록 노드)를
중심으로, 인용된 `Anomaly`(빨강)·`Observation`(주황)·위성/기상/뉴스 노드가 방사형으로 펼쳐진
그래프. 청록 점선은 `correlated_with`(교차 상관) 엣지, 실선은 `cites`/`evidenced_by`/`involves`
등 인용·근거 엣지 — 하단 범례가 노드 색과 엣지 종류를 함께 설명합니다. 이 그래프는 "Q&A가 아니라
결정 루프"라는 프로젝트 원칙의 근거 역추적 가능성을 한 화면에 압축한 것입니다.

## 09. 소스 패널 · 기상 · 뉴스 (우측 열 확대)

![](screens/09_sources_news.png)

온톨로지로 융합된 4개 소스(ADS-B 항적·Celestrak 위성 통과·METAR 기상·GDELT 뉴스)의 객체 카운트,
그리고 **adsb.fi**(교차확인·군용 식별 보강 데이터 제공처) 크레딧 링크. 기상 카드는 RKSI 공항의
임무기상(MVFR·제한) 실황, 뉴스 카드는 발행일(07-02 22:16)과 신뢰도(0.35, 저신뢰 OSINT 표기)가
함께 붙은 `NewsEvent` 객체입니다 — 뉴스는 항상 저신뢰로 표기되며 단독으로 판정 근거가 되지 않습니다.

## 10. 이상징후 결정 루프 — confirm 후

![](screens/10_confirm_loop.png)

03번 화면과 같은 CES7500 이상징후를 사람이 **확인(confirm)**한 직후 상태. 타임라인 배지가
"후보"(주황)에서 **"확인됨"**(빨강)으로 바뀌고, 상세 패널 하단의 확인/기각 버튼이 비활성화되며
"상태: 확인됨" 문구가 남습니다. 이 상태 전이는 서버에 영속되는 `Anomaly` 객체의 액션 — 코파일럿이
답하고 끝나는 게 아니라 사람의 승인이 실제 상태를 바꾸는 **결정 루프(human-on-the-loop)**의
증거입니다.
