## 심각한 오류 기록 (재발 방지)

1. **dropout 오탐 폭주 → 폴러 마비** (2026-07-06, 상세 `docs/worklog/dropout-semantics.md`): gap 임계 90s 고정 × has_gap 이력 고착 × 10분 버킷 재발화 = 시간당 수백 건 오탐 생성, 사이클이 수 분으로 늘며 폴러 사실상 정지. 의미 재정의("지금 침묵"만 발화)로 해소. 교훈: **파생 신호의 임계는 수집 주기와 결합 검증**해야 하고, "이력 보유"와 "현재 상태"를 구분해야 한다.
2. **:8000 고아 프로세스 점유** (2026-07-06): `demo.sh stop`이 pid 파일 기준이라 크래시 라운드를 거친 고아 서버를 못 죽임 → 새 서버는 조용히 기동 실패, 응답은 **삭제된 옛 DB 핸들을 문 고아**가 계속 함(SQLite inode 유지) → "API는 200인데 데이터가 낡은" 유령 상태. 수정: stop()에 포트·패턴 고아 스윕 추가(2026-07-08). 교훈: pid 파일 기반 정지는 크래시 경로에서 반드시 새고, **응답이 있다 ≠ 올바른 프로세스가 응답한다**.
3. **이상징후 무한 누적** (2026-07-06, 상세 `docs/worklog/anomaly-lifecycle.md`): 닫는 경로가 사람 클릭뿐이라 무인 운영에서 candidate 영구 증가(시간당 ~78건). status `resolved`(반증 증거 기반 자동 해소) + 경계이탈 억제로 해소. 교훈: 무인 파이프라인의 산출물엔 **생성 규칙만큼 소멸 규칙**이 필요하다.
