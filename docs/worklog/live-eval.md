## 1. 왜 라이브 P/R은 안 하는가 (정직)

라이브 데이터에는 **ground truth 라벨이 없다**. 어떤 항적이 "진짜 이상"인지 판정할 정답지가
없으므로 precision/recall을 원리적으로 낼 수 없다. 합성 시나리오는 라벨을 우리가 부여하므로 P/R이
산출되지만 그건 "우리가 정한 정답을 우리 룰이 맞히는가"의 준-동어반복(회귀 검증)이지 실세계 성능이
아니다(OVERSOLD#4). 따라서 이 하네스는 **P/R을 흉내내지 않고**, 라이브에서 관측 가능한 3가지만 잰다:

| 측정 항목 | 무엇을 재나 | 기대 | 왜 정직한 신호인가 |
|---|---|---|---|
| **citation 정합** | 라이브 질의의 전 문장 cites 보유율 + cite id 실객체 해상율 | 100% | "무근거 문장 거부·근거 역추적"이 라이브에서도 지켜짐의 실측 |
| **탐지 결정성·분포** | 같은 스냅샷에 룰 2회 → 동일 후보 + 실제 발화 이상징후 유형·건수 | 동일 | 재현성 + "라이브에서 뭐가 잡히나"의 사실 기록(0건이면 0건) |
| **맨몸 LLM 대비** | 라이브 질의 1개의 기계검증 출처 수(파이프라인 vs `claude -p`) | 파이프 다수·맨몸 0 | provenance 유무 = GPT-wrapper 대비 진짜 해자(정확도 경쟁 아님) |

## 2. 신설물

- **`eval/live_eval.py`** — 재실행 가능한 라이브 평가 하네스. `.venv/bin/python -m eval.live_eval`.
  - 라이브 db가 비었으면(리셋 직후) `connectors.opensky.run_poller`를 재사용해 유한 사이클 선수집
    (기본 2사이클, 크레딧 최소 — 뉴스 커넥터 등 **무수정**, 호출만).
  - **스냅샷 격리**: 라이브 db를 sqlite 온라인 백업으로 임시 사본에 복제하고 평가는 사본에 돌린다.
    assess/correlate가 쓰는 SituationAssessment·correlated_with가 런타임 skai.db(실데이터 전용,
    db-regime.md §1)를 오염시키지 않는다. 임시 사본은 실행 끝에 정리 + 다음 실행 시작에 잔재 회수.
  - 산출: `docs/worklog/live_eval_result.json` + stdout 마크다운 표.
- **`tests/test_live_eval.py`** — 네트워크-무관 단위 검증 5건(합성 시나리오로 채운 인메모리 스토어에
  하네스 순수 로직 적용): citation 비율 100%·탐지 결정성(비자명)·분포 정합·assess 재현성·스냅샷 복제.

## 3. 결과 (라이브 스냅샷 2026-07-04 14:11 UTC)

수집: OpenSky+GDELT+METAR+Celestrak 2사이클. 스냅샷 = **항적 42·관측 74·위성통과 93·기상 1(RKSI VFR)
·뉴스 0(GDELT 429 레이트리밋 — 우회 안 함)·이상징후 0**.

### 3-1. citation 정합 — **문장 cites 100% · 해상 100%**
- 질의 10건(의도 다양: situation_summary·count·filter·weather·news) 중 근거 있는 응답 8건, 산출 문장 24개.
- **문장 cites 보유율: 24/24 = 100%** (cites 없는 문장은 write가 거부 = 구조 강제 실측).
- **cite id 실객체 해상율: 98/98 = 100%** (전 인용 id가 실 온톨로지 객체로 역추적).
- no_evidence 2건(군용 추정·뉴스)은 라이브가 얇아 해당 객체 부재 → "해당 없음"을 정직 보고(문장 0,
  비율 분모에서 자연 제외). 즉 **라이브 데이터에서도 "근거 있으면 전부 인용, 없으면 침묵"이 성립**.

### 3-2. 탐지 결정성 + 라이브 분포
- **결정성: 룰 2회 실행 후보 집합 동일**(run1 0건 = run2 0건 — 무작위·벽시계 의존 없음).
- **재현성(보강): 같은 스냅샷·질의 assess 2회 산출 동일**(문장 3개·cites·신뢰도 완전 일치, template
  백엔드). 이상징후 0건이어도 42 실항적으로 코파일럿 read/조립 경로의 결정성을 실측 — "같은 스냅샷
  → 같은 답".
- **라이브 발화 이상징후 총 0건**(비상 스쿽·dropout·로이터링·군용기·위성 근접 모두 0). **정직한 0**:
  이 수집창의 KADIZ 실교통에 발화 조건이 부재했다(비상 스쿽 없음, 민감구역 gap 교차확인 없음, 위성
  통과 93건은 전부 앵커보다 15분+ 미래라 창내 근접 아님). 데모 재현성은 합성 주입기가 담당(설계대로).
- dropout 교차상태: 라이브 2차 피드(adsb.fi) 미배선 → 발화 시 미확인·저신뢰만(단정 금지 코드화).

### 3-3. 맨몸 LLM 대비 (라이브 데이터)
질의 "지금 KADIZ 근방 상황 요약해줘"를 (a)파이프라인 (b)`claude -p` 단독에 던짐:

| 항목 | 온톨로지+파이프라인 | 맨몸 LLM(claude -p) |
|---|---|---|
| 기계검증 출처 수 | **10건**(문장 cites→실 객체 역추적) | **0건** |
| 인용 문장 | 3/3 | 구조적 인용 없음(자유 텍스트) |

- 맨몸 LLM은 **추론은 유능**했다(42 항적 분포·비상 스쿽 부재·서해 특이 항적을 정확히 짚음). 그럼에도
  **기계검증 가능한 출처 0**. 파이프라인은 같은 라이브 데이터에서 10개 실 객체 id를 역추적 가능하게
  인용. 차이의 본질은 정확도 경쟁이 아니라 **provenance**.
- 주의(정직): 중첩 세션에서 `claude -p`는 간헐 타임아웃(60s)한다(run_eval p5에서도 동일 관측). 성공/
  타임아웃 어느 쪽이든 결론(맨몸 기계검증 출처 0)은 불변 — 하네스가 폴백해 구조 비교로 대체한다.

## 4. 발표 수치 프레임 초안 (3수치, 정직)

기존 발표는 "합성 P/R 100%"의 그림자가 남아 있었다. 대체 프레임:

1. **합성 회귀 전건 통과** — 라벨된 자작 시나리오 12건에 탐지 5종 전부 통과 = **탐지 로직 결정성 검증**
   (실세계 정확도 주장 아님).
2. **라이브 citation 해상 100%** — 라이브 데이터 질의에서 문장 cites 24/24·cite 해상 98/98 = 100%.
   "근거 강제가 라이브에서도 지켜진다"의 실측.
3. **라이브 탐지 결정성** — 같은 라이브 스냅샷에 룰 2회 동일 + assess 2회 동일 = 재현성.
   \+ **맨몸 대비 라이브 10 : 0**(파이프라인 기계검증 출처 10 vs 맨몸 LLM 0).

한 줄: **"합성 회귀 전건 통과 + 라이브 citation 해상 100% + 라이브 탐지 결정성, 그리고 맨몸 LLM 대비
기계검증 출처 10:0 — 라이브 P/R은 ground truth가 없어 내세우지 않습니다."**
