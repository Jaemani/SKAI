# direction.md — 백본 5요소 (Air ISR Fusion Copilot)

프로젝트가 흔들릴 때 돌아오는 기준점. 데모·피칭·기능결정 전부 이 5개에 정렬.

---

## 1. 문제 (Problem)
공중·우주 상황 인식은 **항공기 항적 · 위성 궤도 · 기상 · 뉴스/OSINT**가 서로 다른 공개 소스에 흩어져 있어 수작업 융합이 느리고 놓침이 많다.
데이터 양이 부족한 게 아니라 **연결·의미도출이 병목**. 분석가가 창을 5개 띄워놓고 눈으로 교차하는 상황.

핵심 통증:
- 이상징후(비정상 항적, 접근 이벤트, ADS-B 소실)를 사람이 실시간으로 못 따라감.
- 왜 이상한지 **근거(출처)** 없이 "느낌"으로 보고 → 신뢰·검증 불가.
- 지역 단위 상황을 한 화면에 못 모음.

## 2. 사용자 (User)
1차: **공중감시·ISR 분석가 / 당직 상황장교**. 좁은 지역·짧은 시간창을 감시.
2차: 방공 지휘결심 지원(요약 브리핑 소비자).
사용 맥락: "지금부터 30분, 이 반경에서 이상한 거 짚어줘."

## 3. 데이터 (Data — 전부 공개·합법)
- **항적**: OpenSky Network API(무료, ADS-B), ADS-B Exchange.
- **위성 궤도**: Celestrak TLE, Space-Track.org(무료 가입), N2YO.
- **기상**: aviationweather.gov(METAR/TAF), Open-Meteo, NOAA.
- **뉴스/OSINT**: GDELT, NewsAPI, RSS, 웹검색. (필요시 StealthMole은 특수상황 트랙에 우선)
> 상세 엔드포인트·인증·리밋 → `data-sources.md`

## 4. 산출 (Output)
자연어 질의 → **출처가 달린** 지역 상황 답변. 3요소 고정:
1. **상황 요약** (텍스트, 문장마다 citation).
2. **이상징후 리스트** (유형·신뢰도·근거 소스·시각).
3. **시각화**: 지도(항적/위성 지상궤적/지오펜스) + 타임라인.
환각 방지: 근거 소스 ID 없는 주장은 답변에 못 넣는다(citation 강제). 신뢰도(0~1) 표기.

## 5. 차별성 / 방어선 (Moat)
- **온톨로지 척추 (AIP-spine)**: 앱이 아니라 도메인 객체·관계·액션 모델. AIP 에이전트가 그 위에서 추론. GPT-wrapper 챗봇과의 근본 해자. → `ontology.md`
- **맨몸 챗봇 아님**: 라이브 다중소스 융합 + 이상탐지 룰 + AIP Logic 추론 + citation 강제 워크플로.
- **결정 루프**: Q&A가 아니라 근거 인용→결정 제안→사람 승인→Action. 분석가 OODA 단축.
- **비자명 이상징후**: 단순 "비행기 있다"가 아니라 ADS-B dropout(고의 소등 의심), 비상 스쿽(7500/7600/7700), 로이터링, 위성 근접(conjunction), ADIZ 접근.
- **출처 추적(provenance)**: 모든 결론이 온톨로지 근거 객체로 역추적됨 → 군·정보 현장 신뢰 요건 충족.
- **자율 반복**: 라이브 API에 에이전트가 스스로 붙어 검증·개선.
